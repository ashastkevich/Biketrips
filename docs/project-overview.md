# BikeTrips: краткое описание проекта

BikeTrips - сервис для поиска, организации и участия в совместных велопоездках. Проект ориентирован на Россию и страны СНГ, а первая версия задумана как mobile-first web-интерфейс с поддержкой Telegram-бота.

Главная идея продукта - заменить разрозненные объявления о поездках в Telegram-чатах, VK-группах, личных каналах и таблицах единым местом, где у каждой поездки есть понятная карточка, актуальная ссылка, маршрут, список участников и уведомления об изменениях.

Ключевые аудитории:

- организаторы велопоездок, клубы, тренеры, магазины и локальные лидеры сообществ;
- участники-любители и новички, которым важно быстро понять, подходит ли поездка по темпу, сложности, маршруту и формату;
- спонтанные райдеры, ищущие поездку сегодня, завтра или в ближайшие выходные.

MVP должен позволять организатору быстро создать поездку, получить публичную ссылку, управлять участниками и отправлять обновления. Участник должен видеть ближайшие поездки, фильтровать их по городу, дате, типу велосипеда, темпу и уровню, открывать карточку поездки и записываться без сложной регистрации.

Важные сущности проекта:

- лента ближайших поездок;
- карточка поездки с темпом, уровнем, дистанцией, маршрутом, покрытием, форматом `drop/no-drop`, правилами и снаряжением;
- форма создания поездки;
- запись участника через Telegram или веб-форму;
- управление списком участников, лимитом мест и листом ожидания;
- Telegram-бот для подтверждений, напоминаний и уведомлений.

Текущий репозиторий содержит PRD `biketrips-prd-v1.md`, статический прототип главной страницы `index.html` и изображения в директории `img`. При дальнейшей разработке PRD стоит считать основным источником продуктовых требований, этот файл - коротким контекстом для быстрого входа в проект, а `docs/architecture.md` - источником принятых технических решений по стеку и архитектуре.
